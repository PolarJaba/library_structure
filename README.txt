ЛОГИЧЕСКАЯ СТРУКТУРА БАЗЫ ДАННЫХ БИБЛИОТЕКИ

СУЩНОСТИ:

1. Читатель
2. Книга (в фонде)
3. Автор
4. Издательство

СВЯЗИ:

Связь Читатель-Книга осуществляется чеез дополниельную таблицу "Книга на руках", содержащюю 
сопоставление "шифр книги-номер билета" (фактически два вторичных ключей). Такая связь позволяет 
связать сущности Читатель и Книга через таблицу с заранее известным и фиксированным количеством 
столбцов, вместо добавленния неизветсного количесва столбцов "книга" к сущности Читатель или
столбцов "читатель" к Книге. Связи Читатель-[Книга на руках] и [Книга в фонде]-[Книга на руках]
- связи "один ко многим"

Аналогичная схема использована для связи сущностей Автор-Книга (через таблицу "авторство"). 

Между таблицами Издательство-[Книга в фонде] предусмотрена связь "один ко многим".

КЛЮЧИ (по таблицам):

Читатель: первичный ключ "Номер билета (id)".
Книга на руках: сопоставление вторичных ключей "Шифр книги" (вторичный ключ для [Книга в фонде]) с 
"Номер билета (id)" (вторичный ключ для Читатель).
Книга в фонде: первичный ключ "Шифр книги", вторичный ключ для связи с таблицей Издательство 
"Издательство".
Авторство: сопоставление вторичных ключей таблиц Автор-Книга.
Автор: первичный ключ "id автора".
Издательство: первичный ключ "Издательство".
